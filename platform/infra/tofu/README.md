# QASIC Engineering-as-Code – OpenTofu IaC

OpenTofu provisions **cloud** resources only (AWS RDS, ElastiCache, EKS). **Local** development runs via Docker Compose using the repo **Makefile** or Compose directly—not via OpenTofu.

## Local stack (no OpenTofu)

From the **repo root**:

```bash
# Full stack (API, Celery, Redis, Postgres, InfluxDB, MLflow, Grafana)
make run-local
make down-local

# Or without make:
docker compose -f docker-compose.full.yml up -d --build
docker compose -f docker-compose.full.yml down
```

Core stack only (no Influx/MLflow/Grafana): `make run-local-core` or `docker compose up -d --build`.

See the main [README.md](../README.md) for URLs (frontend, API, MLflow, Grafana).

## Prerequisites

- **OpenTofu:** [Install OpenTofu](https://opentofu.org/docs/intro/install/) (1.6+) or Terraform 1.6+
- **AWS:** When using `deployment_target = "aws"` — AWS CLI configured (`aws configure`)

## Push to cloud (AWS)

Provision **RDS (PostgreSQL)**, **ElastiCache (Redis)**, and optionally **EKS** so you can run the Helm chart against managed DBs and cluster. The variable `deployment_target` also accepts `gcp`, `azure`, and `opennebula` for future use; today only `aws` provisions resources. For GKE, AKS, or OpenNebula OneKE, use an existing cluster and deploy with the same Helm chart (see [deploy/README.md](../deploy/README.md)).

1. Apply (password is optional; Tofu can generate and store in Secrets Manager):

   ```bash
   cd infra/tofu
   tofu init
   tofu apply -var="deployment_target=aws"
   # Or pass a password: -var="db_password=YourSecurePassword"
   ```

2. After apply, wire your app or Helm to the outputs:

   ```bash
   tofu output -json
   # Use: aws_database_url, aws_celery_broker_url, eks_cluster_name (if EKS enabled)
   ```

3. For Kubernetes: configure kubeconfig and deploy the Helm chart:

   ```bash
   aws eks update-kubeconfig --region <region> --name $(tofu output -raw eks_cluster_name)
   helm upgrade --install qasic deploy/helm/qasic -f deploy/helm/qasic/values.yaml
   ```

Database credentials can be generated by Tofu and stored in AWS Secrets Manager; use External Secrets Operator in the cluster to sync them into Kubernetes. InfluxDB and MLflow are not created in AWS by this module.

## Layout

```
infra/tofu/
├── versions.tf      # OpenTofu 1.6+, providers (aws, random)
├── variables.tf     # deployment_target, aws_*, db_*
├── locals.tf        # used by AWS modules
├── main.tf          # aws_networking, aws_data, aws_eks (when aws)
├── outputs.tf       # aws_database_url, aws_celery_broker_url, eks_*, etc.
├── README.md
└── modules/
    ├── aws_networking/   # VPC, subnets, security group for RDS/Redis
    ├── aws_data/         # RDS PostgreSQL, ElastiCache Redis, Secrets Manager
    └── aws_eks/          # EKS cluster and node group (optional)
```

## Backend (optional)

To keep state in S3 (e.g. for team or CI), configure the `backend "s3"` block in `versions.tf`, then run `tofu init -reconfigure` and `tofu apply` as usual.
